2 Policies

The following policies are followed by this ontology

  1. All terms within the ontology are typed as skos:Concepts irrespective of their own ontological structures.
  2. Because of the CWRC ontology status as both an ontological construct and a dataset of instances, it is defined as both an OWL ontology and a VOID dataset.
  3. Deprecated or disused terms within the ontology are never removed outright from the ontology file but marked as such in both machine and human readable formats. Whenever possible machine readable equivalences are listed.
  4. To facilitate widespread use of the ontology, instances of an owl class are both typed as such as well as encoded as skos:Concepts that are part of a skos:ConceptScheme.

3 Personal Names

The CWRC makes use of foaf:Person as its basic ontological representation for people. Arguably, foaf is the most successful representation for persons in current use and this ontology is in line with this practice. However, the limited properties supplied and the ambiguity produced in different culture to represent the actual name of the person remain unresolved. Furthermore, as the CWRC deals with many concurrent resources that represent the same entity in different ways, the simple foaf:name property was judged insufficient.

The CWRC ontology provides a number of name related classes that can be used to represent the different names of a person according to different sources:Name at Birth, Maiden Name, Legal Name and Pen Name. These classes extend the skos-xl W3 ontology and enable the use of the provenance ontology to link the name to other source datasets.
